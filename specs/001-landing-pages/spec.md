# Feature Specification: 스토리텔링 진입 및 대화형 입력

**Feature Branch**: `001-landing-pages`  
**Created**: 2026-05-15  
**Priority**: P0 (모든 분석의 진입점)  
**Status**: Active  

---

## Overview

사용자가 서비스에 처음 진입할 때 5단계 스토리텔링 랜딩 페이지(LandingPages)를 통해 서비스의 가치를 경험하고, 이후 SSAju 챗봇과 대화하며(ChatInput) 생년월일과 태어난 시간을 입력한 뒤, 4가지 서비스 선택 카드(ServiceSelect)에서 원하는 분석을 선택하는 전체 진입 흐름입니다.

**Why this Priority**:
- 모든 분석 서비스(관운/컨설팅/기업궁합/피드백)의 유일한 진입점
- 기존 서비스별 개별 입력 폼을 단일 ChatInput 흐름으로 통합 — 사용자는 생년월일/시간을 한 번만 입력
- **P0**: 이 흐름 없이는 어떤 분석 서비스도 시작 불가

## User Story 0 - 스토리텔링 진입 및 대화형 입력

### Independent Test

사용자가 루트(`/`) 접속 → 5단계 랜딩 페이지를 마우스 휠(또는 스와이프/키보드)로 탐색 → 마지막 페이지(Page 5)에서 "내 별자리 보러 가기" 버튼 클릭 → SSAju 챗봇이 단계적으로 메시지를 보내며 생년월일(date picker)과 시간(time picker + 빠른 입력 칩)을 입력받고 → "서비스 고르러 가기" 버튼 클릭 시 4개 서비스 카드 화면 표시 → 카드 클릭 시 해당 서비스 분석 흐름으로 진입.

### Acceptance Scenarios

#### [LandingPages — 5단계 스토리텔링]

1. **Given** 사용자가 루트(`/`) 접속 **When** 페이지 로드 **Then**
   - 별이 빛나는 밤하늘 배경 위에 Page 1이 표시됨
   - 화면 하단에 5개의 페이지 점 인디케이터 표시, 현재 위치(Page 1) 강조
   - "SSAju · 별이 빛나는 밤, 당신의 길을 묻다" 레이블과 "취업 고민, 많으시죠?" 헤드라인 표시
   - 하단에 "다음 페이지" 스크롤 힌트 표시

2. **Given** 랜딩 화면 표시 중 **When** 마우스 휠 아래 스크롤 OR 터치 스와이프 위 OR `ArrowDown`/`PageDown`/`Space` 키 입력 **Then**
   - 다음 페이지(Page 2)로 전환됨
   - 900ms 동안 추가 전환 입력 무시(잠금)
   - 반대 방향 입력 시 이전 페이지로 이동
   - 첫 페이지에서 이전 방향 입력, 마지막 페이지에서 다음 방향 입력은 무시

3. **Given** 5개 페이지를 순서대로 탐색 **When** 각 페이지 표시 **Then** 다음 내용이 표시됨:
   - **Page 1**: 도입부 — "취업 고민, 많으시죠?" 헤드라인 + 감성적 부제
   - **Page 2**: 4가지 취준 고민 카드 그리드 (예: "언제 지원하면 될까요?" 등)
   - **Page 3**: 서비스 소개 — "저희가 풀어드릴게요" + 사주 × AI 결합 소개
   - **Page 4**: 4가지 서비스 소개 카드 (관운 분석 / AI 커리어 컨설팅 / 기업 궁합 분석 / 만족도 피드백)
   - **Page 5**: CTA — "당신의 별, 읽어드릴게요" 헤드라인 + "내 별자리 보러 가기 →" 버튼

4. **Given** 하단 점 인디케이터 클릭 **When** 특정 페이지 번호 점 클릭 **Then** 해당 페이지로 즉시 이동하고 인디케이터 강조 위치 업데이트

#### [ChatInput — 대화형 생년월일/시간 입력]

5. **Given** Page 5에서 "내 별자리 보러 가기" 버튼 클릭 **When** 화면 전환 **Then**
   - 챗봇 대화 인터페이스로 전환
   - 순차 봇 메시지 표시:
     - 400ms 후: "안녕하세요. 저는 SSAju예요. ✨"
     - 이후: "당신의 별자리를 함께 읽어보고 싶어요."
     - 이후: "먼저, 태어나신 날짜를 알려주실래요?" (날짜 입력 UI 포함)

6. **Given** 봇의 날짜 입력 요청 메시지 표시 **When** date picker에서 날짜를 선택하고 → 전송 버튼 클릭 **Then**
   - 사용자 버블(오른쪽)에 선택된 날짜가 "YYYY년 M월 D일" 형식으로 표시
   - 봇 타이핑 인디케이터(…) 0.6초 표시 후 사라짐
   - 1.5초 후 봇 메시지: "좋아요. 이제 태어난 시간도 알려주세요." (시간 입력 UI 포함)

7. **Given** 봇의 시간 입력 요청 메시지 표시 **When** 시간 입력 UI 확인 **Then**
   - HH:mm 형식의 time picker (기본값 12:00) 및 → 전송 버튼 표시
   - 바로 아래 빠른 입력 칩 4개: **00:00**, **06:00**, **12:00**, **18:00**
   - 칩 클릭 시 time picker에 해당 시간 즉시 반영

8. **Given** 시간 입력 후 → 전송 버튼 클릭 **When** 봇 완료 응답 **Then**
   - 사용자 버블에 입력된 시간(HH:mm) 표시
   - 봇 타이핑 인디케이터(…) 0.6초 표시
   - 이후 봇: "고마워요. 별을 펼쳐드릴게요. 🌌"
   - 1.7초 후 봇: "어떤 별자리부터 보여드릴까요?" + **"서비스 고르러 가기 →"** 버튼

9. **Given** 생년월일만 입력 후 시간을 12:00(기본값)으로 유지한 채 전송 **When** 분석 진행 **Then** 시간이 미상인 경우 정오(12:00)으로 분석됨을 봇 메시지에서 안내

#### [ServiceSelect — 서비스 선택 카드]

10. **Given** ChatInput에서 "서비스 고르러 가기" 버튼 클릭 **When** 화면 전환 **Then**
    - "STEP 02 · 서비스 선택" 레이블 표시
    - "어떤 별자리부터 보여드릴까요?" 제목
    - ChatInput에서 입력된 생년월일 · 시간이 금색으로 강조 표시
    - 4개 서비스 카드 그리드 표시:
      - **01 관운 분석** (Career Timing) — 예상 소요 3-5초
      - **02 AI 커리어 컨설팅** (Consultation) — 예상 소요 15-20초
      - **03 기업 궁합 분석** (Compatibility) — 예상 소요 5-8초
      - **04 만족도 피드백** (Feedback) — 예상 소요 1-2초

11. **Given** 데스크톱 환경에서 서비스 카드에 마우스 오버 **When** 마우스 이동 **Then**
    - 카드 내 마우스 커서 위치에 따라 부드러운 광원(라이트 이펙트)이 이동
    - CSS custom property `--mx`, `--my` 값이 마우스 좌표로 실시간 업데이트되어 `radial-gradient` 광원 효과 생성
    - 마우스가 카드 밖으로 벗어나면 광원 효과 사라짐

12. **Given** 서비스 카드 클릭 **When** 선택 완료 **Then**
    - **관운 분석 / AI 커리어 컨설팅**: 로딩 화면 → 분석 결과 화면으로 전환
    - **기업 궁합 분석**: 기업명 및 설립일 추가 입력 화면으로 전환
    - **만족도 피드백**: 만족도 피드백 입력 화면으로 직접 전환

13. **Given** 서비스 선택 화면 표시 **When** "처음으로" 뒤로 가기 버튼 클릭 **Then** ChatInput 화면으로 복귀 (입력된 생년월일/시간 초기화 또는 유지)

## Key Entities

**UserInput** (임시 저장, sessionStorage)
```
{
  birthDate: string (YYYY-MM-DD),
  birthTime: string (HH:mm, 기본값: 12:00),
  selectedService: 'CAREER' | 'CONSULTATION' | 'COMPANY' | 'FEEDBACK' | null
}
```

## Success Criteria

| 기준 | 측정 방법 |
|------|---------|
| S-001 | 페이지 로드 후 3초 이내 첫 번째 랜딩 페이지 표시 |
| S-002 | 페이지 전환 시간 900ms ±50ms (애니메이션 포함) |
| S-003 | 생년월일과 시간 입력 후 5초 이내 서비스 선택 화면 표시 |
| S-004 | 비로그인 사용자도 입력 가능 (저장 불가) |
| UX-001 | 사용자가 힌트 텍스트 없이 인터페이스 조작 가능 |

## Assumptions

- 사용자는 생년월일을 YYYY-MM-DD 형식으로 입력할 수 있음
- 시간이 미상인 경우 12:00(정오)으로 자동 처리
- 모든 브라우저가 `date` 및 `time` input type 지원
- sessionStorage 가능한 환경

---

## Edge Cases

- **Case 1**: 생년월일 형식 오류 (예: 10-10-1990)
  - **Action**: 입력 필드 강조, 에러 메시지 표시
- **Case 2**: 너무 미래 또는 과거의 날짜
  - **Action**: 유효한 범위 확인, 안내 메시지
- **Case 3**: 페이지 전환 중 새로고침
  - **Action**: 진행 상태 sessionStorage에 저장, 복구
- **Case 4**: 서비스 선택 후 취소
  - **Action**: ServiceSelect로 복귀, 입력값 유지
