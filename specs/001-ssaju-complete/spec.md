## Clarifications

### Session 2026-05-11 (Clarification Round 2)

- **Q1: HttpOnly Cookie + sessionStorage 이중 저장 모델**
  - **결정**: HttpOnly + Authorization 헤더 선택
  - **구현 영향**: 
    - HttpOnly Cookie: 백엔드가 설정, CSRF 보호, 모든 요청 자동 포함
    - sessionStorage: 제거 (XSS 위험성)
    - Authorization 헤더: 개발 환경에서만 선택사항 (로깅/디버깅용)
  - **관련 요구사항**: FR-001 (OAuth 로그인), FR-006 (토큰 저장)

- **Q2: 회사명 자동완성 데이터 소스**
  - **결정**: 백엔드 `/api/company/autocomplete` 엔드포인트
  - **구현 영향**:
    - 새 엔드포인트 추가: `POST /api/company/autocomplete` (5s timeout)
    - 요청: `{ query: string }`
    - 응답: `{ suggestions: string[] }`
  - **관련 요구사항**: FR-029 (기업 자동완성)

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 소셜 로그인 및 회원 관리 (Priority: P1)

사용자가 카카오 또는 구글 소셜 로그인을 통해 계정을 연결하고, 분석 결과를 영구적으로 저장하여 나중에 다시 확인할 수 있는 경험.

**Why this priority**:
- 분석 결과 저장 및 히스토리 조회의 핵심 기반
- 사용자가 분석 결과의 영속성을 기대하는 필수 기능
- P1: 마이페이지 기능의 전제 조건

**Independent Test**:
사용자가 로그인 버튼 클릭 → 카카오/구글 팝업 선택 → 소셜 계정 인증 → 프론트엔드가 토큰 수신 및 저장 → 이후 분석 시 저장 버튼 활성화되고, 저장된 결과가 마이페이지에 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 비로그인 상태 사용자 **When** 헤더의 "로그인" 버튼 클릭 **Then** "카카오로 계속하기" 또는 "구글로 계속하기" 옵션 표시

2. **Given** 카카오 로그인 선택 **When** 카카오 인증 팝업에서 승인 **Then** 팝업 닫히고 사용자가 자동으로 로그인됨, 헤더에 "프로필" 또는 "내 분석" 메뉴로 변경

3. **Given** 로그인된 상태 **When** 분석 완료 후 결과 화면 **Then** 결과 하단에 "이 결과 저장하기" 버튼 활성화되어 클릭 시 분석 기록이 백엔드에 저장

4. **Given** 비로그인 상태 분석 진행 **When** 분석 결과 표시 **Then** "결과를 저장하려면 로그인해주세요" 안내문과 "로그인하기" 버튼 표시

5. **Given** 로그인 후 결과 저장 **When** "저장 완료" 메시지 표시 **Then** 해당 분석 기록이 마이페이지에서 즉시 조회 가능

6. **Given** 비로그인 상태 분석 완료 후 페이지 하단에서 휘발성 경고 메시지 확인 **When** 소셜 로그인 버튼(카카오/구글) 클릭하여 인증 완료 **Then** 현재 분석 결과가 백엔드에 자동으로 저장되고 "분석 결과가 저장되었습니다" 메시지 표시, 마이페이지에서 즉시 조회 가능

### User Story 2 - 마이페이지 및 분석 히스토리 조회 (Priority: P1)

로그인한 사용자가 과거에 수행한 모든 분석 기록(관운, 컨설팅, 궁합)을 한눈에 볼 수 있으며, 각 기록을 클릭하면 해당 시점의 분석 결과 화면을 완전히 재현하여 다시 확인할 수 있는 경험.

**Why this priority**:
- 사용자가 과거 분석 결과를 언제든지 참조할 수 있는 핵심 가치
- 분석 이력 관리 및 비교 분석의 기초
- P1: 로그인 기능과 함께 결합되어 서비스의 근본적인 가치 제공

**Independent Test**:
로그인된 사용자 → 마이페이지 접속 → 3가지 분석 타입별 기록 리스트 표시(관운/컨설팅/궁합) → 특정 기록 클릭 → 해당 시점의 완전한 분석 결과 재현(로딩 없이 즉시, 같은 데이터/탭/레이아웃) → 삭제 버튼 클릭 시 기록 삭제 확인.

**Acceptance Scenarios**:

1. **Given** 로그인된 사용자가 마이페이지(`/my-page`) 접속 **When** 페이지 로드 **Then** "내 분석 기록" 제목 + 3가지 탭 표시:
   - 관운 분석 기록
   - AI 컨설팅 기록
   - 기업 궁합 기록

2. **Given** 각 탭 선택 **When** 기록 로드 **Then** 카드 형식으로 기록들 표시:
   - 분석 일시 (예: "2026-05-07 14:30")
   - 분석 대상 정보 (예: "1990-10-10 생년월일")
   - 핵심 결과 미리보기 (예: "H1 권장, 신뢰도 85%")
   - 클릭 가능 상태 표시

3. **Given** 기록 카드 클릭 **When** 상세 페이지 전환 **Then** 해당 분석 결과 화면이 그대로 재현됨:
   - 저장된 데이터가 동일한 레이아웃으로 표시
   - 탭 네비게이션(컨설팅의 경우) 동일하게 작동
   - 로딩 과정 없이 즉시(0.1초 이내) 표시

4. **Given** 재현된 분석 결과 화면 **When** 상단 "뒤로 가기" 또는 "히스토리로 돌아가기" 버튼 클릭 **Then** 마이페이지로 복귀

5. **Given** 기록 카드의 "삭제" 아이콘 클릭 **When** 확인 모달 표시 **Then** "정말 삭제하시겠습니까?" 메시지 + "삭제" / "취소" 버튼

6. **Given** 삭제 확인 **When** "삭제" 클릭 **Then** 해당 기록이 마이페이지에서 즉시 제거되고, 백엔드에서도 삭제 확인

7. **Given** 기록이 없을 때 **When** 마이페이지 탭 확인 **Then** "아직 분석 기록이 없습니다. 지금 분석을 시작해보세요!" 메시지 + "분석하기" 버튼 표시

### User Story 3 - 관운 기반 채용 시기 분석 (Priority: P1)

사용자가 자신의 생년월일과 태어난 시간을 입력하여, 상반기(H1) 또는 하반기(H2) 중 언제가 채용 운이 좋은 시기인지 분석받는 경험.

**Why this priority**: 
- 프로젝트의 핵심 기능이며, 다른 모든 기능의 진입점
- 사용자가 가장 먼저 경험하는 기능
- SajuResult ID를 생성하여 이후 다른 분석들과 연결되는 기초 기능

**Independent Test**: 
사용자가 올바른 생년월일(YYYY-MM-DD)과 시간(HH:mm)을 입력하고 "분석하기"를 클릭했을 때, 3-5초 후 H1 또는 H2 예측 결과와 신뢰도 점수(0-100), 분석 근거가 화면에 표시되며, SajuResult ID가 세션에 저장되는지 확인.

**Acceptance Scenarios**:

1. **Given** 사용자가 관운 분석 화면에 접속 **When** 생년월일 "1990-10-10"과 시간 "14:30"을 입력하고 "분석하기" 버튼을 클릭 **Then** 
   - 고지 문구 "본 사주는 재미로 보는 것이니 참고만 바랍니다"가 페이드 인 오버레이로 강조 표시
   - 정확히 1.5초 후 자동으로 로딩 상태(진행 바)로 전환
   - 3-5초 로딩 후 H1 또는 H2 결과와 신뢰도 프로그레스 바(75% 예시)가 표시, 분석 근거(텍스트) 함께 제시

2. **Given** 사용자가 태어난 시간을 모를 때 **When** 시간 필드에 값을 입력하지 않거나 비워두고 "분석하기" 클릭 **Then** 시간이 미상인 경우 기본값으로 12:00(정오)이 자동 적용되며 분석 진행

3. **Given** 분석 결과가 표시된 후 **When** 결과 하단의 "이 결과에 대해 의견을 알려주세요" 버튼을 클릭 **Then** 피드백 모달이 열리고 만족/불만족 선택 및 의견 입력이 가능

4. **Given** 사용자가 생년월일 형식이 잘못된 값(예: "10-10-1990")을 입력 **When** "분석하기" 클릭 **Then** "생년월일 형식이 올바르지 않습니다 (YYYY-MM-DD)" 메시지 표시, 입력 필드 강조

5. **Given** 사용자가 "분석하기" 버튼 클릭 **When** 고지 문구 "본 사주는 재미로 보는 것이니 참고만 바랍니다"가 화면에 강조 노출 **Then** 별도의 확인 버튼 없이 정확히 1.5초(±50ms) 동안 예쁘게 표시된 후 자동으로 실제 로딩 상태(진행 바)로 자연스럽게 전환됨. 사용자는 강제로 진행하거나 취소할 수 없으며, 자동 전환만 가능

### User Story 4 - AI 기반 커리어 컨설팅 (Priority: P1)

사용자가 관운 분석과 동일한 생년월일/시간을 기반으로, 대규모의 AI 커리어 컨설팅(19개 필드)을 탭 기반 UI로 확인하는 경험.

**Why this priority**:
- 추천 산업, 면접 팁, 사주 프로필, 월별 운세 등 사용자의 커리어 의사결정에 직접 영향
- 높은 가치의 AI 분석 결과 제시 (OpenAI 기반)
- P1: 관운 분석 후 자연스러운 다음 스텝

**Independent Test** (Phase 1 - 8개 탭):
사용자가 생년월일/시간을 입력하고 "AI 컨설팅 받기" 클릭 후, 15-20초의 로딩 메시지("AI 분석 중입니다...")를 본 뒤, 다음 **8개 탭**이 표시되고, 각 탭 클릭 시 해당 데이터가 즉시 전환되는지 확인:
1. 추천산업
2. 면접팁
3. 강점
4. 사주프로필
5. 부의운
6. 경력로드맵
7. 브랜딩
8. 월별운세

> **Phase 2에서 추가**: 정신 관리, 직장 환경 탭 (별도 기능 구현)

**Acceptance Scenarios**:

1. **Given** 관운 분석 결과 화면 **When** "더 알아보기" 또는 "AI 커리어 컨설팅" 버튼 클릭 **Then** 로딩 스피너 표시 + "AI 분석 중입니다 (예상 시간: 15-20초)" 메시지

2. **Given** 15-20초 로딩이 완료 **When** API 응답 수신 **Then** **Phase 1 (8개 탭)**이 표시됨:
   - 추천산업 (상위 2-3개 산업명, 이유, 추천 직무 리스트)
   - 면접팁 (3-5개 항목)
   - 강점 (3-5개 항목)
   - 사주프로필 (일주, 성격 특성, 오행 분포, 십신 분포)
   - 부의운 (소득원, 재무 조언, 투자 성향, 추가 수입)
   - 경력로드맵 (0-2년, 3-5년, 최종목표)
   - 브랜딩 (정장 색상, 이미지, 헤어메이크업, 파워 키워드 3-5개)
   - 월별운세 (1년 12개월의 운세 점수, 행운/주의 분류, 조언)
   
   > **Phase 2 추가 탭**: 정신 관리 (스트레스 관리, 재충전 방법, 마인드셋), 직장 환경 (업무 환경 적응, 업무 스타일, 관계 전략)

3. **Given** 탭이 모두 로드됨 **When** 사용자가 "사주프로필" 탭 클릭 **Then** 즉시(0.2초 이내) 탭 전환되고 일주 천간(예: 丙), 성격 설명, 오행 막대 차트(木火土金水 개수 시각화), 오행 분석 텍스트, 십신 분포가 표시됨

4. **Given** 월별운세 탭 표시 **When** 탭 내용 로드 **Then** 캘린더 형식으로 다음과 같이 표시:
   - 데스크톱: 12개월이 가로로 배치되어 한눈에 조회 가능
   - 모바일: 월별 카드 또는 3개월×4줄 격자로 자동 조정
   - 각 달마다 점수(0-100), 별점(★), 조언 문구 표시
   - 행운의 달: 금색/노란색 배경 + ★ 아이콘
   - 주의의 달: 빨간색 배경 + ⚠️ 아이콘
   - 일반: 회색/중립 배경
   - **달력 네비게이션**: 이전/다음 월 버튼 또는 월 선택 드롭다운으로 특정 월 탐색 가능

5. **Given** 모든 탭 로드 완료 **When** 페이지 스크롤 하단 도달 **Then** "이 분석에 대해 피드백하기" 버튼 표시, 클릭 시 피드백 모달 열림

6. **Given** 로딩 중 20초 초과 경과 **When** 타임아웃 발생 **Then** "AI 분석이 지연되고 있습니다. 잠시 후 다시 시도해주세요." 메시지 + "다시 시도" 버튼 표시, 사용자 클릭 시 최대 3회까지 자동 재시도

### User Story 5 - 기업 궁합 분석 (Priority: P2)

사용자가 자신의 생년월일/시간과 특정 기업의 설립일/시간을 입력하여, 기업과의 궁합 점수 및 직무별 매칭도를 분석받는 경험.

**Why this priority**:
- 취업 준비생이 특정 회사 지원 전 실제 적합성을 판단하는 데 도움
- 기업 정보 자동 조회 실패 시 수동 입력 폴백 제공으로 사용자 편의성 향상
- P2: 선택적 추가 기능이지만 고가치

**Independent Test**:
사용자가 생년월일/시간 입력 후 기업명 "Samsung Electronics" 입력 → API에서 설립일 자동 조회 성공 시 5-8초 후 78/100 점수, 신뢰도(HIGH), 십신/오행/지장간/리더십 점수 분석, 직무별 3-5개 카드(각 점수+이유+추천), 시너지/주의사항, 월별 운세 캘린더가 모두 표시되는지 확인. 기업 조회 실패 시 "설립일 수동 입력" 폴백이 제시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 기업 궁합 분석 화면 **When** 사용자 생년월일 "1990-10-10", 시간 "14:30", 기업명 "삼성전자" 입력 후 "궁합 분석하기" 클릭 **Then** 5-8초 로딩 후 종합 점수(예: 78/100)와 신뢰도(LOW/MEDIUM/HIGH) 표시

2. **Given** 분석 결과 표시 **When** 점수 영역 확인 **Then** 점수 분석 항목 4개(십신 궁합, 오행 궁합, 지장간 궁합, 리더십 매칭)가 각각 점수 + 진행 막대로 시각화

### User Story 6 - 만족도 피드백 (Priority: P1)

사용자가 각 분석 결과에 대해 만족도(만족/불만족)와 상세 의견(최대 500자)을 모달 형태로 제출하는 경험.

**Why this priority**:
- 모든 분석 결과(관운/컨설팅/기업궁합)와 연결되는 필수 기능
- 사용자 피드백 수집으로 서비스 개선의 기초
- P1: 각 분석 화면마다 피드백 버튼 필수 배치

**Independent Test**:
사용자가 분석 결과 화면 하단의 "피드백 제출" 버튼 클릭 → 모달이 열리고 "만족함/만족하지 않음" 라디오 버튼 표시 → 피드백 유형 자동 선택됨(분석 화면에 따라 CAREER_TIMING/CONSULTATION/COMPATIBILITY) → 텍스트 영역에 최대 500자 의견 입력 가능 → "제출하기" 클릭 시 1-2초 후 "피드백이 저장되었습니다" 메시지 표시 및 모달 닫힘, 세션에 저장된 SajuResult ID와 함께 백엔드로 전송 확인.

**Acceptance Scenarios**:

1. **Given** 관운 분석 결과 화면 **When** 결과 하단 "이 결과에 대해 의견을 알려주세요" 버튼 클릭 **Then** 모달 열림, 제목 "분석 결과 피드백", 닫기(×) 버튼 표시

2. **Given** 피드백 모달 열림 **When** 모달 내용 확인 **Then** 다음 필드 표시:
   - 만족도 선택 (필수): ○ 만족함, ○ 만족하지 않음 (라디오 버튼)
   - 피드백 유형: 화면에 따라 자동 선택됨 (읽기 전용 또는 자동 입력)
   - 상세 의견 (선택): 텍스트 영역, 최대 500자, 글자 수 카운터 표시(예: "120 / 500")

3. **Given** 사용자가 "만족함" 선택 **When** 모달 내 확인 **Then** 라디오 버튼 선택 상태 즉시 반영

4. **Given** 상세 의견 텍스트 영역에 입력 **When** 350자 입력 완료 **Then** 카운터가 "350 / 500"으로 실시간 업데이트

5. **Given** 의견 500자 도달 **When** 501자 입력 시도 **Then** 501번째 글자 입력 방지, 경고 메시지 없이 자동 제한

6. **Given** 만족도 선택 후 **When** "제출하기" 버튼 클릭 **Then** 로딩 스피너 표시, 1-2초 후 성공 메시지 "피드백이 저장되었습니다.", 모달 자동 닫힘

7. **Given** 피드백 제출 중 네트워크 오류 **When** 3초 초과 응답 없음 **Then** "제출 중 오류가 발생했습니다. 다시 시도해주세요." 메시지 + "다시 시도" 버튼

8. **Given** 사용자가 피드백을 제출하지 않고 닫기(×) 버튼 클릭 **When** 모달 닫힘 **Then** 입력된 데이터 저장 안 함, 경고 메시지 없음

### Edge Cases

- **시간 미상 처리**: 사용자가 생년월일만 입력하고 시간을 빠뜨렸을 때, 자동으로 12:00(정오)을 기본값으로 적용하되, UI에서 명시적으로 "시간이 미상인 경우 정오(12:00)로 분석됩니다" 안내 표시
  
- **기업 정보 조회 실패**: 공공데이터 API 조회 실패 시 수동 입력 폴백이 자연스럽게 제시되어야 함. 사용자가 "설립일/시간 모름" 옵션 선택 시 기본값 적용(설립시간: 12:00)

- **타임아웃 및 재시도** (백그라운드 자동 처리, 사용자에게 숨김):
  - **CareerTiming**: 3-5초 타임아웃 시 최대 3회 자동 재시도 (각 3초 간격) → 최악 9초 추가 대기
  - **Consultation**: 20초 타임아웃 시 최대 2회 자동 재시도 (각 5초 간격) → 최악 30초 추가 대기
  - **Compatibility**: 8초 타임아웃 시 최대 3회 자동 재시도 (각 2초 간격) → 최악 6초 추가 대기
  - **UI 처리**: 재시도는 모두 백그라운드에서 자동 처리. 사용자에게는 "로딩 중...", "AI 분석 중입니다 (예상 시간: ~초)", "분석 중..." 등 **단일 메시지만 표시**. 재시도 횟수나 시도 번호는 노출하지 않음
  - **모든 재시도 실패 시**: "분석에 실패했습니다. 잠시 후 다시 시도해주세요" + "다시 시도" 버튼

- **비로그인 사용자의 휘발성 데이터 관리**: 비로그인 사용자의 분석 결과는 현재 페이지의 메모리(Zustand/State)에만 유지되며, 다음 상황에서 즉시 파기됨:
  - 페이지 새로고침 시
  - 브라우저 탭 종료 시
  - 다른 페이지로 이동 시
  - 브라우저 종료 시
  - 분석 결과를 저장하려면 반드시 로그인 필수. 비로그인 상태에서는 결과 저장 불가능
  - **회원 전용 가치**: 분석 결과의 영구 저장은 로그인 사용자만 가능하며, 마이페이지에서 언제든지 히스토리 조회 가능

- **세션 만료**: 페이지 새로고침 후에도 `sessionStorage`에서 `sajuResultId` 복원되어 이전 분석 결과와 연결된 피드백 제출 가능

- **잘못된 입력 형식**: 
  - 날짜 형식 오류(예: "10/10/1990", "2000-13-32") → "생년월일은 YYYY-MM-DD 형식입니다"
  - 시간 형식 오류(예: "25:00", "14-30") → "시간은 HH:mm 형식입니다 (예: 14:30)"

- **브라우저 뒤로 가기**: 분석 결과 화면에서 뒤로 가기 시 입력 폼 상태 유지 또는 초기 상태로 복원 (UX 선택 사항이지만, 사용자 편의 고려)

- **소셜 로그인 토큰 만료**: 로그인된 사용자의 토큰이 만료되면 자동으로 로그아웃 처리 → "로그인이 만료되었습니다. 다시 로그인해주세요" 안내

- **중복 분석 저장**: 같은 생년월일/기업으로 여러 번 분석을 수행했을 때, 각 분석이 별도로 저장되며 모두 히스토리에 표시됨

- **마이페이지 데이터 로딩 실패**: 히스토리 조회 중 백엔드 오류 시 "분석 기록을 불러올 수 없습니다" 메시지 + "다시 시도" 버튼

- **결과 재현 시 저장된 데이터 누락**: 저장된 분석 결과 중 일부 필드가 손상되었을 경우, 손상된 필드는 "데이터 없음" 표시 + 기록 재분석 옵션 제시

- **세션 중 탭/메뉴 전환**: 로딩 중인 분석이 있을 때 다른 페이지로 이동하면, 이전 분석이 백그라운드에서 계속 진행되다가 완료 후 자동으로 해당 페이지로 복귀 또는 팝업 알림

- **모바일 환경에서 소셜 로그인**: 모바일 브라우저에서는 리다이렉트 방식을 사용하여 팝업 차단 문제 회피. 카카오/구글 인증 완료 후 앱으로 자동 복귀하고, 로그인 토큰 저장 후 분석 기능 활성화

- **소셜 로그인 실패**: 로그인 과정 중 네트워크 오류, API 다운, 사용자 취소 등으로 실패하면, 오류 메시지 표시 후 비로그인으로 계속 분석 가능. 현재 분석 결과는 Zustand 메모리에 유지되며, 나중에 로그인하면 자동 저장 가능 (localStorage 미사용 — FR-022)

## Requirements *(mandatory)*

### Functional Requirements

#### 1. 입력 검증 및 데이터 처리

- **FR-001**: 시스템은 생년월일을 `YYYY-MM-DD` 형식으로만 수락. 다른 형식(슬래시, 점 등)은 거절하고 "생년월일은 YYYY-MM-DD 형식입니다" 메시지 표시

- **FR-002**: 시스템은 태어난 시간을 `HH:mm` (24시간 기준) 형식으로만 수락. 범위는 `00:00` ~ `23:59`

- **FR-003**: 시간 필드가 비어있거나 입력되지 않을 경우, 자동으로 `12:00` (정오)을 기본값으로 적용. UI에는 "시간이 미상인 경우 정오(12:00)로 분석합니다" 안내 문구 표시

- **FR-004**: 생년월일은 과거 날짜만 수락. 미래 날짜 입력 시 "생년월일은 오늘 이전이어야 합니다" 메시지 표시

- **FR-005**: 시스템은 입력된 모든 텍스트(기업명, 피드백 의견 등)에 대해 HTML/스크립트 자동 이스케이프 처리(XSS 방지)
ㅁ
#### 2. API 호출 및 응답 처리

- **FR-006**: 관운 분석 API 호출 시 요청 본문은 `{ "birthDate": "1990-10-10", "birthTime": "14:30", "solarType": "SOLAR" }` 형식. (`solarType`: 양력/음력 구분, 기본값 "SOLAR"). 응답은 `sajuResultId`, `h1Period`, `h2Period`, `h1Confidence` (0-100), `h2Confidence` (0-100), `recommendation` (문자열) 포함

- **FR-007**: AI 커리어 컨설팅 API는 **한 번의 POST 호출로 8개 탭 데이터를 모두 수신** (Q5 결정 — Phase 1 MVP). 예상 응답 시간은 15-20초. 이 시간 동안 로딩 스피너 + "AI 분석 중입니다..." 메시지 표시. 응답 수신 후 모든 탭 데이터가 로드되어 탭 클릭 시 즉시(0.2초 이내) 전환 가능.
  - **Phase 1 탭 (8개)**: 추천산업, 면접팁, 강점, 사주프로필, 부의운, 경력로드맵, 브랜딩, 월별운세
  - **Phase 2 추가 탭**: 정신관리(`mentalCare`), 직장환경(`environmentFit`, `workStyle`, `relationshipStrategy`)

- **FR-008**: 기업 궁합 분석 API는 다음 우선순위로 기업 정보 조회:
  1. 기업명 제공 시 → 공공데이터 API로 자동 조회
  2. 조회 성공 시 → 확인 모달에서 조회 결과(설립일/시간) 표시, 사용자가 "확인" 또는 "수정" 선택
  3. 조회 실패 시 → "설립일을 수동으로 입력해주세요" 안내
  4. 사용자가 설립일 수동 입력 시 → 입력값 사용

- **FR-009**: 피드백 API 호출 시 요청 본문에 `sajuResultId` (세션에서 관리), `feedbackType` (자동 결정), `satisfactionStatus` (사용자 선택), `feedbackContent` (선택 항목, 최대 500자) 포함

- **FR-010**: 모든 API 응답은 공통 형식 준수: `{ "success": boolean, "data": {...}, "error": {...}, "timestamp": number }`

#### 3. 소셜 로그인 및 인증 관리

- **FR-011**: 사용자가 헤더의 "로그인" 버튼을 통해 카카오(Kakao) 또는 구글(Google) 소셜 로그인 선택 가능
  - **데스크톱/태블릿**: OAuth 팝업 방식으로 인증 진행 (사용자 맥락 유지)
  - **모바일**: OAuth 리다이렉트 방식으로 인증 진행 (팝업 차단 회피, 완료 후 앱으로 자동 복귀)

- **FR-012**: 소셜 로그인 성공 시 백엔드에서 반환된 인증 토큰을 클라이언트에 안전하게 저장하여 이후 API 요청에 포함

- **FR-013**: 로그인 상태는 페이지 로드 시 자동 복원되어 새로고침 후에도 로그인 유지 (단, 토큰 만료 시 자동 로그아웃)

- **FR-014**: 비로그인 사용자도 모든 분석을 수행할 수 있으나, 분석 결과를 **저장하려면 반드시 로그인 필수**. 비로그인 상태에서 분석 완료 시 "결과를 저장하려면 로그인해주세요" 안내 제시

- **FR-015**: 로그아웃 시 저장된 토큰 및 로컬 사용자 정보 완전히 삭제, 세션 초기화

- **FR-016**: 소셜 로그인 실패 시 (네트워크 오류, API 다운, 사용자 취소 등):
  - 사용자 친화적 오류 메시지 표시 (예: "카카오 로그인에 실패했습니다. 네트워크를 확인하고 다시 시도해주세요")
  - "다시 시도" 버튼 제공 (최대 3회 재시도 권장)
  - 비로그인으로 계속 분석 가능 (로그인 강제 안 함)
  - 로그인 실패해도 현재 분석 데이터(Zustand 메모리)는 유지 (localStorage 미사용 — FR-022)

#### 4. 데이터 저장 및 세션 관리

- **FR-016a**: 로그인한 사용자의 분석 결과는 "이 결과 저장하기" 버튼 클릭 시 백엔드에 영구 저장되며, 저장 완료 메시지 표시

- **FR-017**: 저장된 분석 결과는 마이페이지(`/my-page`)에서 분석 타입별(관운/컨설팅/궁합)로 리스트 표시, 분석 날시와 핵심 정보 미리보기 포함

- **FR-018**: 마이페이지의 저장된 기록 클릭 시 해당 시점의 분석 결과가 **로딩 없이 즉시(0.1초 이내)** 완전히 재현되어 표시

- **FR-019**: 마이페이지에서 개별 기록의 "삭제" 버튼으로 불필요한 기록 삭제 가능, 삭제 후 백엔드 동기화

- **FR-020**: 분석 결과 화면 도출 후 백엔드에서 반환된 `SajuResult ID`를 `sessionStorage`에 저장하여 피드백 제출 시 함께 전송

- **FR-021**: **로그인한 사용자 vs 비로그인 사용자의 데이터 관리 전략**
  - **로그인한 사용자**: 
    - `sessionStorage`: 현재 분석 세션 관리 (`sajuResultId`, `lastAnalysisType`)
    - **백엔드 DB**: 분석 결과 **영구 저장** (마이페이지에서 조회 시 API 호출로 데이터 조회, 로그인 필수)
    - **분석 결과의 영구 저장 및 마이페이지 히스토리 조회는 오직 회원에게만 제공되는 핵심 가치**
  - **비로그인 사용자**: 
    - 분석 결과는 **현재 페이지의 메모리(Zustand/State)에만 유지**
    - **저장 불가**: 페이지 새로고침이나 이탈 시 즉시 파기됨 (영구 저장 불가, localStorage 사용 안 함)
    - 비로그인 사용자는 **마이페이지 접근 불가** (로그인 필수)

- **FR-022**: **비로그인 사용자의 세션 격리 및 휘발성 보장**
  - **현재 페이지만 메모리 유지**: 비로그인 사용자의 분석 결과는 **Zustand 상태 관리**로 현재 페이지의 메모리에만 유지
  - **즉시 파기**: 페이지 새로고침, 브라우저 탭 종료, 또는 다른 페이지로 이동 시 **즉시 삭제**
  - **로그인 필수 저장**: 분석 결과를 보관하고 나중에 조회하려면 반드시 로그인하여 백엔드에 저장
  - **보안**: 탭 닫으면 해당 탭의 분석 데이터 즉시 삭제

- **FR-023**: **저장소 정보 유지**
  - **sessionStorage** (로그인/비로그인 공통): `sajuResultId`, `lastAnalysisType` (CAREER_TIMING/CONSULTATION/COMPATIBILITY)
  - **HttpOnly Cookie** (로그인 시만): 인증 토큰 — 백엔드가 설정, 클라이언트에서 직접 접근 불가 (Q1 결정: XSS 방지)
  - **메모리(Zustand)** (비로그인): 현재 분석 결과 데이터 (페이지 새로고침 시 초기화)
  - **백엔드 DB** (로그인): 마이페이지 조회용 분석 기록 (영구 저장)

#### 5. 비로그인 사용자의 휘발성 데이터 경고 및 가입 유도

- **FR-024**: **휘발성 데이터 경고 및 가입 유도 메시지**
  - **트리거**: 비로그인 사용자가 분석 결과 페이지를 **스크롤하여 viewport 맨 아래에 도달**할 때 표시
    - **구현**: Intersection Observer API로 마지막 콘텐츠 요소 감지하여 자동 표시
    - **타이밍**: 사용자가 분석 결과를 충분히 검토한 후 자연스럽게 가입 유도
  - **메시지 내용**:
    - **텍스트**: "로그인하지 않으면 이 결과는 페이지를 나갈 때 사라집니다. 영구 보관하시겠습니까?"
    - **CTA**: 메시지 하단에 **소셜 로그인 버튼** (카카오, 구글) 배치
    - **시각적 강조**: 배경색(주의색 예: 주황/노랑)이나 테두리로 구분하여 사용자의 주목 유도
  - **로그인 완료 후 자동 저장**: 비로그인 사용자가 로그인 버튼을 클릭하여 소셜 인증을 완료하면:
    - 현재 분석 결과가 백엔드에 **자동으로 저장**됨 (사용자 명시적 저장 버튼 클릭 불필요)
    - "분석 결과가 저장되었습니다" 메시지 표시
    - 저장된 기록이 마이페이지에서 즉시 조회 가능
  - **로그인 사용자**: 이 메시지 미표시 (분석 결과가 명시적 저장 버튼으로 제어됨)
  - **핵심**: 분석 결과의 영구 저장 및 마이페이지 히스토리 조회는 오직 회원에게만 제공되는 가치임을 명시

- **FR-025**: **페이지 이탈 방지 컨펌 모달** (비로그인 사용자 전용)
  - **트리거**: 비로그인 사용자가 분석 결과 화면에서 **페이지 이탈 시에만** 모달 표시
    - 뒤로가기 (브라우저 백 버튼 또는 "뒤로 가기" 링크)
    - 다른 분석 화면으로 이동 (새로운 분석 시작)
    - 외부 링크 또는 메뉴 클릭으로 다른 페이지로 이동
  - **제외**: 새로고침(F5), 탭 닫기는 포함하지 않음 (사용자 편의성 고려)
  - **컨펌 모달 내용**:
    - **메시지**: "지금 나가시면 분석 결과가 삭제됩니다. 정말 나가시겠습니까?"
    - **버튼**: 
      - "지금 로그인하기" (카카오/구글 소셜 로그인으로 이동) → 로그인 완료 후 **같은 분석 결과 페이지로 자동 복귀**, 분석 결과는 자동으로 저장됨
      - "그냥 나가기" (분석 결과 폐기 후 페이지 이탈)
      - "계속 보기" (모달 닫고 현재 페이지 유지)
  - **로그인 완료 후**: 같은 분석 결과 페이지로 자동 복귀하며, 분석 결과가 이미 백엔드에 저장되었으므로 "분석 결과가 저장되었습니다" 메시지 표시. 마이페이지에서 즉시 조회 가능
  - **로그인 사용자**: 이 모달 미표시 (분석 결과가 저장되므로 경고 불필요)
  - **구현 방식**: `beforeunload` 또는 라우터의 `beforeRouteLeave` 이벤트로 하드 네비게이션 감지
  - **보안**: 이탈 방지 로직은 비로그인 사용자만 적용

#### 6. 디자인 시스템 및 시각 언어 (별이 빛나는 밤 테마)

- **FR-061**: **컬러 팔레트 (별이 빛나는 밤 모티브)**
  - **배경색**: 진한 파란색/자주색 밤하늘 톤 (예: #0A1E3E, #1A0F2E) → 전체 앱 배경
  - **강조색**: 밝은 노란색/금색 (예: #FFD700, #FFA500) → 별 모티브, 중요 요소
  - **중립색**: 흰색 (#FFFFFF), 라이트 그레이 (#F5F5F5) → 텍스트, 카드 배경
  - **보조색**: 은색/라벤더 (예: #C0C0C0, #E6E6FA) → 세부 장식
  - **상태색**: 성공(초록), 경고(주황), 에러(빨강) → 표준 의미론적 색상
  - **별 장식 요소**: 페이지 배경, 로딩 애니메이션, 탭 활성 표시 등에 ★ 아이콘 또는 별 모양 SVG 배치

- **FR-062**: **입력 폼 컴포넌트 디자인**
  - **생년월일 입력**: HTML5 `<input type="date">` 사용하되, 커스텀 스타일링 적용
    - 기본 브라우저 날짜 피커 UI 대신 **아름다운 캘린더 픽커 라이브러리 사용** (예: react-datepicker, react-calendar 등으로 스타일 커스터마이징)
    - 캘린더 픽커: 밤하늘 테마 적용 (배경 #0A1E3E, 날짜 텍스트 흰색, 선택된 날짜 배경 금색)
    - 별 모티브: 오늘 날짜에 ★ 아이콘 표시
  - **시간 입력**: `HH:mm` 텍스트 입력 + 숫자 스피너 또는 타임 피커 (12:00 기본값 안내)
  - **기업명 입력**: 텍스트 입력 + 검색 자동완성 UI (공공데이터 API로 조회 시 후보 표시)
  - **폼 레이아웃**: 세로 배열 (생년월일 → 시간 → 기업명 순서), 모바일에서도 가독성 유지

- **FR-063**: **메시지 및 피드백 UI**
  - **토스트 알림** (비침투적 메시지): 우상단에서 나타났다 자동으로 3초 후 사라짐
    - 성공: 초록 배경, "✓ 분석 결과가 저장되었습니다"
    - 경고: 주황 배경, "⚠️ 로그인 후 저장 가능합니다"
    - 에러: 빨강 배경, "✗ AI 분석 중 오류 발생"
  - **입력 검증 에러**: 각 입력 필드 바로 아래 인라인 에러 메시지 (빨간색 텍스트 14px)
  - **모달 (중요 확인)**: 가운데 배치, 반투명 검정 오버레이 (opacity: 0.6), 배경 흐림 처리 (blur)
    - 예: 페이지 이탈 확인, 기록 삭제 확인, 로그인 모달 등

- **FR-064**: **탭 네비게이션 및 컴포넌트 스타일**
  - **탭 배치**: 수평 가로 배열 (AI 컨설팅의 8개 탭)
    - **데스크톱(1024px+)**: 모든 탭이 한 줄로 표시
    - **태블릿(768px~1023px)**: 스크롤 가능한 탭 네비게이션 (좌우 스크롤)
    - **모바일(360px~430px)**: 탭이 가로로 스크롤 가능하며, 가장자리에 ">" 스크롤 힌트 표시
  - **탭 스타일**: 언더라인 형식 (활성 탭: 금색 언더라인 3px, 비활성 탭: 투명), 호버 시 라벤더 배경
  - **활성 탭 표시**: ★ 별 아이콘 또는 금색 강조
  - **탭 전환**: 0.2초 이내 부드러운 애니메이션 전환

- **FR-065**: **타이포그래피 및 폰트 체계**
  - **기본 폰트**: Pretendard 또는 Noto Sans KR (별이 빛나는 밤 테마와 조화)
  - **추가 폰트** (우아한 느낌): Serif 폰트 1가지 선택
    - 제목/헤더 용도: 우아한 한글 Serif (예: 한글 Garamond, 바탕, 궁서체 현대 버전)
    - 본문: Pretendard (가독성 중심)
  - **폰트 크기 계층**:
    - **페이지 제목** (가장 큰 제목): 데스크톱 32px, 태블릿 28px, 모바일 24px / Serif / Bold / 색상: 금색 (#FFD700)
    - **섹션 제목** (중간 제목): 데스크톱 24px, 태블릿 20px, 모바일 18px / Sans-serif / Semi-bold
    - **본문 텍스트**: 데스크톱 16px, 태블릿 15px, 모바일 14px / Sans-serif / Regular / 색상: 흰색 또는 라이트 그레이
    - **라벨/캡션**: 14px / Sans-serif / Medium / 색상: 은색 (#C0C0C0)
  - **라인 높이**: 1.6 (가독성 확보), 글자 간격: 0.5px (우아함)

#### 7. 마이페이지 및 분석 히스토리 조회 (로그인 사용자 전용)

- **FR-034**: `/my-page` 라우트에서 로그인한 사용자만 접근 가능. 비로그인 사용자는 로그인 페이지로 리다이렉트
  - **핵심 가치**: 분석 결과의 영구 저장 및 마이페이지 히스토리 조회는 오직 회원에게만 제공됨을 명시

- **FR-035**: 마이페이지는 3가지 탭으로 분류된 분석 기록 표시:
  - 관운 분석 (CareerTiming)
  - AI 컨설팅 (Consultation)
  - 기업 궁합 (Compatibility)

- **FR-036**: 각 기록 카드는 다음 정보 표시: 분석 날시, 분석 대상 정보(생년월일 또는 기업명), 핵심 결과 미리보기(예: "H1 권장 신뢰도 85%")

- **FR-037**: 기록 카드 클릭 시 해당 `SajuResult ID`를 기반으로 **백엔드 DB에서 저장된 전체 분석 데이터를 조회** (API 호출). 조회된 데이터를 **원본 분석 결과 페이지와 동일한 레이아웃으로 재현**. 탭 네비게이션, 시각화, 모든 텍스트가 정확히 동일하게 표시되며, 로딩은 0.1초 이내에 완료

- **FR-038**: 재현된 분석 결과에는 "뒤로 가기" 또는 "히스토리로 돌아가기" 버튼 제공하여 마이페이지로 복귀 가능

- **FR-039**: 각 기록에 "삭제" 버튼 제공. 삭제 클릭 시 확인 모달("정말 삭제하시겠습니까?") 표시 후, 승인 시 백엔드에서 삭제 및 마이페이지에서 즉시 제거

- **FR-040**: 분석 기록이 없을 때 "아직 분석 기록이 없습니다. 지금 분석을 시작해보세요!" 메시지 + 분석 페이지로 이동하는 버튼 표시

- **FR-041**: 마이페이지 로드 시 분석 기록을 최신순으로 정렬하여 표시. 초기 로드는 **최신 20개만 fetch**, 사용자가 스크롤하여 하단에 도달하면 다음 20개 자동 로드 (무한 스크롤)

- **FR-042**: 마이페이지 무한 스크롤 로딩 중 하단에 로딩 스피너 표시. 모든 기록을 로드 완료 시 "더 이상 기록이 없습니다" 메시지 표시

#### 8. 반응형 웹 디자인

- **FR-043**: **데스크톱 중심 설계**: 기본 설계 대상은 1024px 이상의 데스크톱 화면. 모바일 우선 설계가 아님을 명시

- **FR-044**: **태블릿 지원(768px ~ 1023px)**: 태블릿 해상도에서 레이아웃이 자동 조정되어 탭 네비게이션, 입력 폼, 결과 표시가 가독성 있게 배치

- **FR-045**: **모바일 지원(360px ~ 430px)**: 스마트폰 환경에서도 탭 네비게이션이 깨지지 않으며, 방대한 분석 결과 텍스트(예: 월별 운세 12개월)가 스크롤 가능한 형태로 가독성 유지. 상세 분석 텍스트는 필요시 펼침/접힘 UI(아코디언)로 제공

- **FR-046**: **터치 친화적 UI**: 모바일/태블릿에서 버튼, 탭, 입력 필드의 최소 터치 영역이 44px 이상으로 설계되어 오터치 방지

- **FR-047**: **이미지 및 차트 반응형**: 월별 운세 캘린더, 오행 분포 차트 등 시각 자료가 화면 크기에 맞춰 자동 축소/확대되며, 모바일에서도 의미 전달이 명확함

- **FR-048**: **폰트 크기 조정**: 데스크톱(16px 본문) → 태블릿(15px) → 모바일(14px)로 단계적으로 조정되어 모든 기기에서 적절한 가독성 제공

- **FR-049**: **CSS Grid/Flexbox 사용**: 모든 레이아웃에 Tailwind CSS의 반응형 유틸리티(sm, md, lg, xl) 활용하여 미디어 쿼리 최소화

- **FR-050**: **모바일에서의 마이페이지**: 기록 카드들이 단일 열 레이아웃으로 표시되어 스크롤하기 편함. 기록 카드의 미리보기 정보가 줄임표(...) 처리되어 넘침 방지

#### 9. 분석 전 고지 및 면책

- **FR-055**: **분석 전 고지 문구 강조 노출 (전환 UI)**
  - **트리거**: 사용자가 "분석하기" 버튼 클릭 시 실제 분석 로딩 진입 전 고지 문구 표시
  - **고지 문구**: "본 사주는 재미로 보는 것이니 참고만 바랍니다"
  - **표현 방식**: 
    - **오버레이 방식**: 화면 전체를 opacity: 0.5의 다크 오버레이로 덮고, 중앙에 흰색 텍스트로 고지 문구를 강조 표시
    - **반응형 폰트 크기**: 데스크톱(28px) → 태블릿(24px) → 모바일(20px)로 기기별 최적화하여 가독성 확보
    - 사용자 입력 차단 (터치/클릭/스크롤 불가능한 상태)
  - **사용자 상호작용**: 
    - 별도의 확인 버튼 없음 (사용자가 버튼을 눌러야 진행되는 방식 제외)
    - 고지 문구는 자동으로 **정확히 1.5초간** 표시됨
    - 사용자가 강제로 진행하거나 취소할 수 없음 (자동 전환만 가능)
  - **접근성**: 스크린 리더 사용자도 고지 문구를 인식할 수 있도록 `role="alert"` 사용

- **FR-056**: **고지 문구에서 로딩 상태로의 자동 전환**
  - **타이밍**: 고지 문구 표시 후 **정확히 1.5초 경과**
  - **애니메이션 상세**: 
    - 고지 문구 화면에 **500ms ease-in-out 페이드 아웃** 효과 적용
    - 동시에 로딩 상태(진행 바)에 **500ms ease-in-out 페이드 인** 효과 적용
    - 두 애니메이션이 동시에 진행되어 자연스러운 전환 표현
  - **로딩 상태**: 고지 문구 전환 후 기존 로딩 UI (진행 바, "로딩 중..." 메시지) 표시
  - **성능 보증**: 페이드 아웃/인 애니메이션은 60fps 이상의 부드러운 프레임 유지로 끊김 없는 전환 보장
  - **취소 불가**: 전환 후에도 사용자가 분석을 취소할 수 없음 (분석 진행 중)

#### 10. 데이터 시각화 및 상태 관리

- **FR-057**: **데이터 시각화 컴포넌트 표준화**
  - **오행 분포 시각화**: 사주 프로필의 오행 분포(木火土金水)는 **막대 차트(Bar Chart)**로 시각화하여 각 오행의 개수를 직관적으로 표현
  - **점수 시각화**: 분석 신뢰도(0-100), 궁합 점수, 십신/오행/지장간/리더십 개별 점수는 **시각적인 진행 바(Progress Bar)**로 표현
  - **월별 운세 시각화**: 12개월의 운세 데이터는 **캘린더 형식**으로 제공하며, 다음 요소 포함:
    - 데스크톱(1024px+): 12개월을 가로로 배치하여 한눈에 조회 가능
    - 모바일(360-430px): 월별 카드 리스트 또는 3개월×4줄 격자 형식으로 자동 조정
    - **달력 네비게이션 UI**: 이전/다음 월 버튼 또는 월 선택 드롭다운으로 사용자가 특정 월을 쉽게 탐색
    - 행운/주의/일반 월을 색상(금색/빨간색/회색) + 아이콘(★/⚠️)으로 시각적 구분
  - 시각화 컴포넌트는 데스크톱/태블릿/모바일 모든 기기에서 반응형으로 표시

- **FR-058**: **간결한 로딩 UX (Loading Experience)**
  - **AI 커리어 컨설팅 호출 시** (예상 대기: 15-20초)
    - 구체적인 예상 대기 시간 텍스트나 카운트다운 UI는 제외 (예: "약 15초 소요"라는 텍스트 미표시)
    - **시각적인 진행 바(Progress Bar)만 노출**하여 분석이 진행 중임을 인지시킴
    - 로딩 바는 **부드러운 애니메이션**을 포함하며, 분석 완료까지 지속적으로 움직임 (무한 루프 애니메이션)
    - UI의 청결함을 유지하면서도 사용자에게 명확한 로딩 신호 전달

- **FR-059**: **적응형 에러 복구 UI (Adaptive Error Recovery)**
  - API 에러 발생 시 단순 경고 메시지만 표시하지 않고, **사용자의 다음 행동을 즉시 유도하는 UI로 전환**
    - **타임아웃 에러** (FASTAPI_TIMEOUT, OPENAI_API_TIMEOUT): "다시 시도" 버튼이 즉시 노출되어 사용자가 클릭하면 자동 재시도 시작
    - **기업 정보 미발견** (COMPANY_NOT_FOUND): "설립일을 수동으로 입력해주세요"라는 안내와 함께 **설립일/시간 입력 폼이 자동으로 활성화**
  - 모든 에러 상태는 **ErrorMessage 컴포넌트 규격**을 따르며, 사용자의 다음 행동을 유도하는 버튼(`onClick` 핸들러 포함) 포함

- **FR-060**: **탭 상태 보존 및 즉시 전환 (Tab Persistence & State Management)**
  - AI 커리어 컨설팅의 방대한 탭 데이터(8개 탭 × 다양한 필드)는 **Zustand 전역 상태 관리**로 구현
  - **데이터 유지**: 한 번 로드된 분석 결과는 메모리에 유지되어 탭 이동 시 재호출 또는 재렌더링 없음
  - **즉시 전환**: 탭 전환 응답성은 **200ms 이하** (0.2초 이내)로 보장
  - **광범위 유지**: 페이지 내 탭 이동뿐 아니라, 다른 메뉴로 이동했다가 복귀 시에도 이전 분석 결과 데이터가 메모리에 유지되어 재호출 없이 즉시 표시
  - **초기화 조건**: 새로운 분석(다른 생년월일/기업명 입력) 시작 시에만 상태 초기화, 로그인/로그아웃 시 초기화

### Key Entities *(Data Model without Implementation Details)*

#### 0. **User**
- 소셜 로그인을 통해 생성된 사용자 프로필
- 속성: `userId` (고유 식별자), `socialProvider` (KAKAO/GOOGLE), `socialId` (소셜 제공자의 사용자 ID), `email` (이메일, 선택사항), `name` (사용자명), `profileImage` (프로필 이미지 URL, 선택사항), `createdAt` (가입 시각)
- 역할: 분석 결과 저장 및 마이페이지 접근 권한 관리

#### 1. **SajuResult** (문서에서는 "`SajuResult ID`"로 통일)
- 한 번의 분석 요청에 대한 고유 ID와 메타데이터
- 속성: `resultId` (Backend: DB PK), `userId` (로그인 시 사용자 ID, 비로그인은 null), `birthDate` (입력 날짜), `birthTime` (입력 시간), `analysisType` (CAREER_TIMING/CONSULTATION/COMPATIBILITY), `isSaved` (저장 여부), `createdAt` (생성 시각), `savedAt` (저장 시각, 저장된 경우만)
- **클라이언트 변수명**: `sajuResultId` (`sessionStorage`에 저장되는 변수 이름)
- 역할: 피드백 제출 시 분석 결과와 피드백 데이터 연결, 마이페이지에서 분석 기록 조회, 저장된 기록 재현 시 API 요청 파라미터로 사용

#### 2. **CareerTimingData**
- 관운 분석 결과
- 속성: `favoredPeriod` (H1/H2), `confidenceScore` (0-100), `reasoning` (분석 근거 텍스트)

#### 3. **ConsultationData**
- AI 커리어 컨설팅 상세 분석 결과
- **Phase 1 (8개 탭, 주요 필드)**:
  - 기본 조언: `industries` (배열), `interviewTips` (배열), `strengths` (배열)
  - 관운: `favoredPeriod`, `confidenceScore`, `reasoning`
  - 사주 프로필: `sajuProfile` (객체, 일주/오행/십신 포함)
  - 재무 스타일: `wealthStyle` (소득원/조언/투자성향)
  - 경력 로드맵: `longTermRoadmap` (단계별 목표)
  - 개인 브랜딩: `personalBranding` (색상/이미지/키워드)
  - 월별 운세: `careerTimeline` (12개월 점수 + 조언)
- **Phase 2 (추가 필드, 별도 구현)**:
  - 정신 관리: `mentalCare` (스트레스/재충전/마인드셋)
  - 직장 환경: `environmentFit`, `workStyle`, `relationshipStrategy`

#### 4. **CompatibilityData**
- 기업/직무 궁합 분석 결과
- 속성: `compatibilityScore` (0-100), `confidenceLevel` (LOW/MEDIUM/HIGH), `reasoning` (종합 평가 텍스트)
- 하위 데이터:
  - `scoreBreakdown` (십신/오행/지장간/리더십 개별 점수)
  - `roleCompatibility` (직무별 3-5개 카드)
  - `synergies` (강점 항목들)
  - `cautions` (주의사항 항목들)
  - `monthlyForecast` (월별 점수/조언, **캘린더 형식**으로 12개월 시각화: 행운/주의/일반 월 구분, 달력 네비게이션 UI 포함)
  - `careerMilestones` (3단계 발전 계획)

#### 5. **UserFeedback**
- 사용자 만족도 피드백
- 속성: `feedbackId` (고유 ID), `sajuResultId` (연결된 분석), `feedbackType` (CAREER_TIMING/CONSULTATION/COMPATIBILITY), `satisfactionStatus` (SATISFIED/DISSATISFIED), `feedbackContent` (최대 500자 텍스트), `createdAt` (제출 시각)

#### 6. **UserInput**
- 사용자가 입력한 기본 정보 (일시적 저장)
- 속성: `birthDate` (YYYY-MM-DD), `birthTime` (HH:mm 또는 기본값 12:00), `companyName` (기업궁합 시 선택), `companyFoundingDate` (기업궁합 시 선택)

## Success Criteria

### Functional Success

- **S-001**: 모든 6가지 사용자 스토리(소셜 로그인/마이페이지/관운/컨설팅/기업궁합/피드백)가 정의된 대로 작동하여 사용자가 완전한 커리어 분석 경험과 히스토리 관리 기능을 얻음
  - **회원 전용 가치**: 분석 결과의 영구 저장 및 마이페이지 히스토리 조회는 오직 회원(로그인 사용자)에게만 제공되어 가입 유도 효과 극대화

- **S-002**: 입력 검증이 모든 형식 오류를 방지하고, 사용자 친화적 오류 메시지로 올바른 재입력 유도

## Assumptions

1. **Backend 가용성**: Spring Boot 백엔드는 `http://localhost:8080` (로컬) 또는 프로덕션 도메인에서 24/7 가용하며, API 응답은 명세된 시간 내 완료

2. **시간 입력 기본값**: 사용자가 시간을 입력하지 않을 경우, 정확한 사주 분석을 위해 12:00(정오)을 기본값으로 적용하는 것이 업계 표준

3. **공공데이터 API**: 기업명 자동 조회 실패 시 사용자가 설립일 수동 입력을 선택할 것으로 예상. 수동 입력 폴백은 완벽한 UX를 위해 필수

4. **네트워크 안정성**: 로컬 개발 환경(localhost)에서는 네트워크 안정적, 프로덕션에서는 CDN/로드밸런싱으로 지원

5. **브라우저 기능**: 모든 최신 주요 브라우저(Chrome/Safari/Firefox/Edge)에서 `sessionStorage` API 지원 가정

6. **사용자 기기**: 데스크톱(1024px+) 중심 설계, 태블릿(768px~1023px) 및 모바일(360px~430px) 반응형 지원. 모바일 우선 설계가 아님

7. **소셜 로그인 제공자**: 카카오 및 구글 소셜 로그인 API가 정상 가용하며, 백엔드에서 토큰 교환 및 사용자 정보 관리 가능

8. **마이페이지 데이터 저장**: 로그인한 사용자가 분석 결과를 저장할 때 전체 분석 데이터가 백엔드에 영구 저장되며, 마이페이지에서 조회 시 원본과 동일한 상태로 재현 가능

9. **세션 캐싱 및 데이터 휘발성**: 
   - **로그인 사용자**: `sessionStorage`에 저장된 분석 결과 데이터를 브라우저에서 즉시 접근, 새로고침 시 초기화. 마이페이지에서 백엔드 DB의 영구 저장본 조회 가능 (회원 전용 가치)
   - **비로그인 사용자**: 분석 결과는 현재 페이지의 메모리(Zustand/State)에만 유지. 페이지 새로고침이나 이탈 시 즉시 파기됨 (localStorage 사용 안 함, 7일 보관 안 함)

10. **비로그인 분석 (Phase 1)**: 비로그인 사용자는 모든 분석을 수행할 수 있으나, 휘발성 메모리에만 저장
    - **핵심**: 비로그인 사용자의 분석 결과는 저장할 수 없으며, 페이지를 떠나면 즉시 삭제됨
    - **회원 전용 가치**: 분석 결과의 영구 저장 및 마이페이지 히스토리 조회는 오직 회원(로그인 사용자)에게만 제공
    - Phase 2에서 비로그인 사용자의 추가 기능 제한 검토 예정

11. **데이터 시각화 라이브러리**: 차트(막대/진행 바)와 타임라인 UI는 Chart.js, Recharts, 또는 유사한 경량 시각화 라이브러리로 구현 가능

12. **상태 관리 라이브러리**: Zustand를 사용하여 전역 상태 관리 (AI 커리어 컨설팅의 8개 탭 데이터 메모리 캐싱)

13. **로딩 애니메이션**: 부드러운 진행 바 애니메이션은 CSS 애니메이션 또는 라이브러리(예: react-loading, nanobar)로 구현 가능

14. **에러 복구 UX**: ErrorMessage 컴포넌트는 에러 코드에 따라 자동으로 "재시도" 또는 "수동입력" UI로 분기 처리

#### Q2: OAuth 팝업 vs 리다이렉트 (태블릿 기기 브레이크포인트)
- **질문**: 태블릿(768-1024px) 기기에서 소셜 로그인 시 팝업을 사용할 것인가, 리다이렉트를 사용할 것인가?
- **선택**: **B - 태블릿도 리다이렉트 (768px 이상 모두 리다이렉트)**
- **상세 설명**:
  - **모바일 (< 768px)**: 팝업 방식 사용 (좁은 화면에서 팝업이 더 적합)
  - **태블릿 (768px - 1024px)**: **리다이렉트** (새 탭/창으로 OAuth 페이지 이동)
  - **데스크톱 (≥ 1024px)**: 리다이렉트 방식 사용
  - `usePlatformDetect()` 로직 업데이트: `window.innerWidth < 768` ← **팝업**, `>= 768px` ← **리다이렉트**
  - 이유: 768px 이상은 충분한 화면 너비가 있어 리다이렉트해도 사용자 경험 저하 없음

#### Q5: API 재시도 정책 상세
- **질문**: API 타임아웃/에러 발생 시 어떻게 재시도할 것인가?
- **선택**: **A - 타임아웃만 재시도 (3회), 지수백오프 1s-2s-4s**
- **상세 설명**:
  - **재시도 대상**: `FASTAPI_TIMEOUT`, `OPENAI_API_TIMEOUT`, `REQUEST_TIMEOUT`, `NETWORK_ERROR` 만
  - **재시도 제외**: `INVALID_REQUEST` (400), `RESOURCE_NOT_FOUND` (404), `INVALID_SAJU_DATA` 등 (사용자 입력 오류)
  - **재시도 횟수**: 최대 3회
  - **재시도 간격** (지수 백오프):
    - 1차 실패 후 1초 대기
    - 2차 실패 후 2초 대기 (1 × 2)
    - 3차 실패 후 4초 대기 (2 × 2)
  - **3회 모두 실패 후**:
    - ErrorMessage 컴포넌트 표시
    - "재시도" 버튼 + "다른 방법 시도" 버튼 제공
    - 사용자가 "재시도" 클릭 시 재시도 횟수 리셋 (다시 3회 가능)
  
  ```typescript
  // 재시도 로직 (plan.md 5.3 apiFetch에 반영)
  const delays = [1000, 2000, 4000]; // 1s, 2s, 4s
  for (let attempt = 0; attempt < 3; attempt++) {
    try {
      // API 호출
    } catch (error) {
      if (isRetryable(error) && attempt < 2) {
        await sleep(delays[attempt]);
        continue;
      }
      throw error;
    }
  }
  ```

- **plan.md 섹션 5.3**: `apiFetch()` 함수의 재시도 로직 구체화
- **ERROR_CONFIG**: 각 에러 코드별 `retryable` 플래그 설정 필요
- **ErrorMessage 컴포넌트**: "재시도" 버튼 로직 (재시도 횟수 리셋)
